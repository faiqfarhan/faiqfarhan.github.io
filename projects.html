selenium-pom-framework

What it is: A production-ready Selenium + Python test framework with Page Object Model, pytest, parallel execution, HTML reports, and GitHub Actions CI — ready to clone and run.

This framework was built from patterns I developed across multiple projects. It solves the most common problems in Selenium projects: flaky locators, poor test isolation, slow sequential execution, and zero visibility in CI. It's the starting point I wish I'd had when I first set up Selenium at Octek.

What's inside
  • Page Object Model structure — every page has its own class with locators and actions cleanly separated
  • pytest fixtures for browser setup/teardown — no boilerplate in test files
  • Parallel test execution with pytest-xdist, cutting suite time significantly
  • HTML report generation with pytest-html — shareable test evidence after every run
  • GitHub Actions workflow — tests run automatically on every push and pull request
  • Sample tests against a live demo site (SauceDemo) so you can see it working immediately
  • Configurable via a single config.py — switch browser, base URL, headless mode in one place
Key code structure
# pages/login_page.py
class LoginPage:
    URL = "https://www.saucedemo.com"
    USERNAME_INPUT = (By.ID, "user-name")
    PASSWORD_INPUT = (By.ID, "password")
    LOGIN_BUTTON   = (By.ID, "login-button")

    def login(self, username, password):
        self.driver.find_element(*self.USERNAME_INPUT).send_keys(username)
        self.driver.find_element(*self.PASSWORD_INPUT).send_keys(password)
        self.driver.find_element(*self.LOGIN_BUTTON).click()

jmeter-load-test-suite

What it is: A complete JMeter test suite with realistic multi-step user journeys, response time assertions, Grafana dashboard config, and a shell script to run everything from CI.

Performance testing is often treated as a one-off activity before a big release. This suite is designed to make it continuous — tests run in CI on every release candidate, and the Grafana dashboard gives instant visibility into regressions in response time or error rate. Built from patterns used at InstaFreight and The XPertz.

What's inside
  • JMeter test plan (.jmx) covering a realistic e-commerce flow: browse → search → add to cart → checkout
  • Parameterised user data via CSV — simulate 500 unique users with different credentials and behaviours
  • Response time SLA assertions — test fails automatically if p95 exceeds threshold
  • InfluxDB + Grafana dashboard config — import one JSON file and get instant metrics visibility
  • run_tests.sh — single command to execute tests, generate reports, and post results to Slack
  • GitHub Actions workflow — runs a smoke load test (50 users, 2 min) on every PR
  • Detailed README with setup instructions, tuning guidance, and how to read the reports
CI execution command
# run_tests.sh — used in GitHub Actions
jmeter -n \
  -t test-plans/ecommerce_load.jmx \
  -l results/results.jtl \
  -e -o results/html-report \
  -Jthreads=100 \
  -Jrampup=60 \
  -Jduration=300 \
  -Jp95_threshold=2000   # fail if p95 > 2s

Logistics API Automation — InstaFreight

Outcome: 30% increase in testing efficiency. Regression suite ran in CI on every deployment, catching integration bugs before they reached staging.

InstaFreight's platform connects freight shippers and carriers through a complex set of REST APIs — shipment creation, pricing, carrier matching, and tracking. Manual API testing was a bottleneck. I built a structured Postman + Newman test suite covering all core endpoints with data-driven scenarios and integrated it into the CI pipeline.

Challenge
  • APIs were tested manually before each release — slow, inconsistent, and error-prone
  • No standard way to share test scenarios across team members or reproduce failures
  • Several edge cases (empty carrier lists, pricing timeouts) were never tested until bugs were found in production
Approach
  • Mapped all API endpoints and defined test scenarios for happy paths and key error conditions
  • Built Postman collections with pre-request scripts for auth token management and environment variables
  • Used data files (CSV) to drive data-variation tests — different shipment sizes, routes, and carrier types
  • Integrated Newman (Postman CLI) into Jenkins pipeline — tests run on every deployment to staging
  • Set up test reports in TestRail, linking automated results to manual test cases for traceability

R&D Verification Framework — Ansys

Outcome: Scalable automated verification framework covering hardware/software integration layers — improved coverage, reliability, and compliance traceability across complex simulation software releases.

At Ansys, software verification must meet stringent quality standards for engineering simulation products used in aerospace, automotive, and industrial applications. I designed and implemented end-to-end verification strategies covering test planning through automated regression, with full traceability to requirements.

Key activities
  • Designed verification plans mapping tests to functional and non-functional requirements for traceability
  • Built automated regression suites for hardware/software integration test scenarios
  • Conducted root cause analysis on failures — distinguishing test environment issues from genuine defects
  • Collaborated with R&D teams to define acceptance criteria for complex simulation features
  • Maintained detailed quality metric reporting for stakeholders and compliance documentation

Mobile & Web Test Automation — Octek

Outcome: 20% improvement in testing efficiency through a new test case management process and partial automation of mobile regression flows.

At Octek I was responsible for testing both web and mobile (iOS and Android) applications. Manual regression was time-consuming and inconsistent. I introduced structured test case management and automated the highest-risk mobile user journeys using Java and JMeter for backend load scenarios.

Approach
  • Designed and documented test cases covering all critical mobile flows for iOS and Android in JIRA
  • Implemented a structured test case management process — priority tagging, owner assignment, and sprint linking
  • Wrote JMeter scripts to stress-test backend APIs serving the mobile applications
  • Collaborated with developers to reproduce and diagnose device-specific defects
  • Ensured on-time delivery by coordinating testing milestones with release schedules
